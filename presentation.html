01 — The Problem

Heart disease is the world's leading cause of death

Early detection dramatically improves outcomes, but diagnosis requires expensive tests, specialist visits, and weeks of waiting — resources many people lack.

❤️

17.9 million deaths/year

Cardiovascular disease kills more people globally than any other condition — accounting for 32% of all deaths worldwide.

⏳

Late diagnosis is fatal

Most patients present to emergency care only after a cardiac event. Proactive, accessible risk screening could save millions of lives.

🏥

Specialist access is limited

Cardiologists are scarce and expensive. In developing countries, the ratio can be 1 specialist per 100,000 people.

📊

Data already exists

Routine blood tests and ECGs capture all the features needed to predict cardiac risk — but the insight isn't being automatically extracted.

CardioScan bridges the gap between existing clinical data and actionable cardiac risk insight — making preliminary screening fast, free, and accessible.

02 — The Solution

From patient data to risk assessment in seconds

CardioScan uses a trained Random Forest model to analyse 13 clinical indicators and deliver an instant, explainable cardiac risk score.

1

Input Data

Patient enters 13 clinical values including age, ECG results, cholesterol, and BP

2

Validation

Live range checking flags abnormal values before submission

3

ML Prediction

Random Forest model evaluates all features simultaneously using ensemble decision trees

4

Risk Score

Probability score (0–100%) displayed with visual gauge and clinical alerts

5

PDF Report

Professional clinical report exported for sharing with physicians

Why Random Forest?

Random Forest is ideal for structured clinical data — it handles missing values gracefully, is resistant to overfitting on small datasets, and naturally outputs feature importance scores that make predictions explainable to healthcare professionals.

Grid search cross-validation was used to find optimal hyperparameters, achieving 88.5% accuracy on the UCI Heart Disease dataset.

03 — Features Built

What CardioScan delivers today

Every feature was deliberately chosen to make the tool useful for real clinical screening scenarios.

🔮

ML Risk Prediction

Random Forest classifier trained on 303 patient records from the UCI Heart Disease dataset. Returns both binary prediction and probability score.

📊

Feature Importance Chart

Interactive Chart.js bar chart showing which of the 13 clinical features most influenced the current prediction — bringing transparency to the AI decision.

💪

BMI Auto-Calculator

Calculates and classifies BMI in real time from height and weight inputs. Adds obesity/overweight clinical alert if BMI ≥ 25.

⚠️

Clinical Alert System

Automatically flags critical values: BP ≥ 180, cholesterol ≥ 240, age ≥ 70, low heart rate, ST depression. Displayed before and after prediction.

📄

PDF Report Export

Generates a professional, formatted clinical report using ReportLab — includes patient details, all vitals, risk score, and clinical alerts.

🕒

Session History

In-session patient history table with risk bar visualisation. Supports up to 50 records per session with one-click clear.

05 — Future Scope & Scalability

How CardioScan can grow

The current build is a solid foundation. Here's the roadmap from student project to real clinical tool.

Version 2 — Multi-model ensemble

Add XGBoost and Logistic Regression as comparison models. Let users see which model predicts what and why — boosting trust and educational value.

Version 3 — Database + user accounts

Replace in-memory history with PostgreSQL. Add physician login to track patients across visits and generate longitudinal risk trend reports.

Version 4 — EHR integration

FHIR-compliant API layer to pull data directly from Electronic Health Record systems. Eliminates manual data entry — the biggest barrier to clinical adoption.

Version 5 — Mobile app + wearables

React Native frontend + Apple Health / Fitbit integration. Continuous passive risk monitoring using real-time heart rate and activity data.

06 — Real-World Impact

Why this matters beyond the grade

CardioScan demonstrates how machine learning can democratise access to specialist-level insight in resource-limited settings.

Rural & underserved areas

A community health worker with a smartphone and basic blood test results could screen dozens of patients per day using CardioScan — no cardiologist needed for a first pass.

Preventive care shift

Moving from reactive emergency treatment to proactive risk screening could reduce cardiac events by 20–30%, according to WHO estimates on early intervention.

Clinical decision support

Even for trained doctors, a second-opinion risk score with feature importance helps prioritise which patients need urgent ECG or stress tests first.

"The model's accuracy of 88.5% on the UCI dataset is competitive with published clinical decision support systems — making this not just an academic exercise, but a genuine proof-of-concept for deployable medical AI."

Ethical considerations

Not a replacement for doctors

CardioScan is a decision support tool — every output is clearly labelled "for educational purposes only" and recommends physician consultation.

Dataset limitations

The UCI dataset is US-demographic and 1988-era. A production system would require diverse, recent data for generalisability.

No data storage

Current version stores no data to disk — session history is in-memory only. This is the right privacy default for a prototype.
